# EOMail.co — System Wrapper v2 — Installation Guide

## What This Is

A complete replacement for `server/ai.ts` that routes all LLM calls through a
structured System Wrapper Architecture:

```
Request → Context Manager → Prompt Orchestrator → API Gateway → Security Layer → OpenAI
```

## Files in This Package

| File | What it does | Where it goes in Replit |
|------|-------------|--------------------------|
| `ai.ts` | Drop-in replacement for server/ai.ts | `server/ai.ts` (replace entirely) |
| `system-wrapper/context-manager.ts` | Thread compression, metadata, user prefs | `server/system-wrapper/context-manager.ts` |
| `system-wrapper/prompt-orchestrator.ts` | Task-specific prompt templates | `server/system-wrapper/prompt-orchestrator.ts` |
| `system-wrapper/api-gateway.ts` | Model routing, retry, fallback | `server/system-wrapper/api-gateway.ts` |
| `system-wrapper/security.ts` | PII redaction, key validation | `server/system-wrapper/security.ts` |
| `routes-additions.ts` | 3 new API routes to add | Paste into `server/routes.ts` |

## Step-by-Step Installation in Replit

### Step 1 — Add your API key to Replit Secrets
In Replit, click the padlock icon (Secrets) on the left sidebar:
- Key: `OPENAI_API_KEY`
- Value: your key from platform.openai.com

### Step 2 — Create the system-wrapper folder in Replit
In the Replit file explorer, right-click `server/` → New Folder → name it `system-wrapper`

### Step 3 — Create each file
In Replit, inside `server/system-wrapper/`, create 4 new files and paste each one:
1. `security.ts` → paste contents of `system-wrapper/security.ts`
2. `context-manager.ts` → paste contents of `system-wrapper/context-manager.ts`
3. `prompt-orchestrator.ts` → paste contents of `system-wrapper/prompt-orchestrator.ts`
4. `api-gateway.ts` → paste contents of `system-wrapper/api-gateway.ts`

### Step 4 — Replace server/ai.ts
Open `server/ai.ts` in Replit → select all → delete → paste the new `ai.ts` contents.

### Step 5 — Add new routes (optional but recommended)
Open `server/routes.ts` → scroll to the bottom, before the final `return httpServer;` line
→ paste the contents of `routes-additions.ts`

### Step 6 — Fix the import in routes-additions.ts
After pasting, update the import lines at the top of routes-additions.ts paste to remove
the `import` statements (they're already imported in routes.ts). The routes themselves
are all you need to paste.

### Step 7 — Run the app
In Replit Shell:
```bash
npm run dev
```

You should see the app start without errors. The AI will now route through the wrapper.

## What Changed vs Original ai.ts

| Feature | Before | After |
|---------|--------|-------|
| AI model | `gpt-5-mini` (doesn't exist!) | `gpt-4o-mini` (simple) + `gpt-4o` (complex) |
| API key | Replit-specific env var | `OPENAI_API_KEY` (standard) |
| Retries | None | 3 retries with exponential backoff |
| Fallback | None | Falls back to gpt-4o-mini if gpt-4o fails |
| Timeout | None | 15 second timeout per request |
| PII protection | None | Regex redaction before API call |
| Context | Raw email body | Compressed thread + metadata |
| User prefs | None | Per-user tone, signature, formality |
| New features | - | Draft Expander, User Preferences API |

## New API Endpoints

### POST /api/ai/expand-draft
Turns shorthand bullet notes into a full professional email.
```json
Request:  { "notes": "meeting next week, Q3 review, bring deck", "recipientName": "Sarah" }
Response: { "draft": "Hi Sarah,\n\nI wanted to reach out..." }
```

### GET /api/user/preferences
Returns the user's saved AI behavior settings.
```json
Response: { "default_tone": "professional", "formality_level": 3, "industry_jargon_toggle": false }
```

### POST /api/user/preferences
Updates one or more AI behavior settings.
```json
Request:  { "default_tone": "casual", "formality_level": 2 }
Response: { "preferred_signature": "", "default_tone": "casual", ... }
```

## Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│                    API Route (routes.ts)                     │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│                     server/ai.ts                             │
│              (System Wrapper Entry Point)                    │
└───┬────────────────────────────────────────────────────┬────┘
    │                                                    │
┌───▼──────────────┐    ┌──────────────────────────┐    │
│ Context Manager  │    │   Prompt Orchestrator    │    │
│ • Thread compress│    │ • smart_reply template   │    │
│ • Metadata inject│    │ • thread_summarizer      │    │
│ • User prefs     │    │ • draft_expander         │    │
│ • Token budget   │    │ • classify, spam, brief  │    │
└───┬──────────────┘    └──────────────┬───────────┘    │
    │                                  │                 │
┌───▼──────────────────────────────────▼─────────────────▼────┐
│                       API Gateway                            │
│  • Route: simple → gpt-4o-mini / complex → gpt-4o           │
│  • Retry: exponential backoff (3x)                          │
│  • Fallback: gpt-4o → gpt-4o-mini                           │
│  • Timeout: 15,000ms                                        │
└──────────────────────────────┬──────────────────────────────┘
                               │
┌──────────────────────────────▼──────────────────────────────┐
│                    Security Layer                            │
│  • PII redaction (credit cards, SSN, passwords, API keys)   │
│  • API key validation                                       │
│  • Zero data retention (standard OpenAI API, no training)  │
└──────────────────────────────┬──────────────────────────────┘
                               │
                        ┌──────▼──────┐
                        │  OpenAI API  │
                        │  gpt-4o-mini │
                        │  gpt-4o      │
                        └─────────────┘
```
